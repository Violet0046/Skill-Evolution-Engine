# SubAgent规则（执行层）

## 职责边界

SubAgent负责**分析和进化**，根据证据自己决策。

### 允许做的事

1. 读取任务文件（包含失败模式和证据）
2. 读取原始SKILL定义
3. 分析失败模式的根因
4. 根据证据生成进化建议
5. 修改SKILL定义，输出SKILL_v2.md

### 禁止做的事

1. **禁止**修改任务文件
2. **禁止**跳过分析直接生成建议
3. **禁止**忽略证据中的关键信息

## 工作流程

```
1. 读取任务文件 tasks/{skill_name}.json
2. 读取原始SKILL定义 skills/{skill_name}/SKILL.md
3. 分析失败模式：
   - 查看每个失败模式的证据
   - 理解失败的上下文（reasoning, input_summary）
   - 识别失败的根本原因
4. 生成进化建议：
   - 基于证据，不是基于猜测
   - 针对具体问题，不是泛泛而谈
5. 修改SKILL定义：
   - 保留原始SKILL的核心逻辑
   - 添加针对性的错误处理
   - 输出到 skills/{skill_name}/SKILL_v2.md
```

## 分析原则

### 证据驱动

- **必须**查看具体的证据（session_id, uuid, error_output）
- **必须**理解失败的上下文（reasoning, input_summary）
- **禁止**基于错误模式名称猜测原因

### 问题导向

- **必须**针对具体问题生成解决方案
- **禁止**生成泛泛的建议（如"检查脚本路径"）
- **必须**提供可操作的改进

### 保持简洁

- **禁止**在SKILL中添加过多的错误处理
- **只**添加针对高频失败的处理
- **保留**原始SKILL的核心逻辑

## 输出格式

进化后的SKILL应该包含：

```markdown
# {skill_name}（进化版）

## 原始内容
（保留原始SKILL的完整内容）

## 针对性改进（基于证据）

### 问题1：{具体问题描述}
- 证据：{引用具体的失败案例}
- 改进：{具体的改进措施}

### 问题2：{具体问题描述}
- 证据：{引用具体的失败案例}
- 改进：{具体的改进措施}

## 进化记录

| 版本 | 日期 | 变更内容 |
|------|------|----------|
| v1.0 | - | 原始版本 |
| v2.0 | {date} | 基于{count}个失败案例进化 |
```
